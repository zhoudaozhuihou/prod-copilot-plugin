# Product Development Workflow

## 1. Diverge

Use `/brainstorm` to generate broad product ideas.

## 2. Converge

Use `/feature` to select one MVP-level feature and make it executable.

## 3. Specify

Use `/prd` to produce a full product requirement document.

## 4. Experience

Use `/journey` to validate the user path and identify friction points.

## 5. Contract

Use `/api` to create API contracts.

## 6. Deliver

Use `/task` and `/test` to prepare implementation.

## 7. Govern

Use `/review`, `/diff`, and `/release` before merging or deployment.
