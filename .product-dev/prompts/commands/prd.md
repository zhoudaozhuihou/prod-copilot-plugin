# PRD Command Prompt

Create or update a Product Requirements Document that is executable by engineers, QA, and autonomous coding agents.

## Required behavior

1. Ask only critical clarifying questions if needed; use lettered options.
2. Define goals, non-goals, users, constraints, success metrics, dependencies, and risks.
3. Produce small user stories suitable for Ralph-style one-iteration execution.
4. Acceptance criteria must be concrete and verifiable.
5. UI stories need visual verification criteria.
6. Data stories need DQ/reconciliation criteria.
7. Code stories need typecheck/lint/test criteria.
8. End with whether `/story-split` or `/prd-json` should run next.
