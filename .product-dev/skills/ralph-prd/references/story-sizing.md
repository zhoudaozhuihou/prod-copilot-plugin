# Ralph Story Sizing

## Right-sized examples

- Add one database column and migration.
- Implement one API endpoint with tests.
- Add one UI component to an existing page.
- Add one filter dropdown and persist state.
- Add one DQ rule and validation SQL.

## Too large; split these

- Build the entire dashboard.
- Add authentication end-to-end.
- Refactor the API layer.
- Migrate all Oracle SQL to BigQuery.
- Build all data quality controls.

## Dependency ordering

1. Schema/data contract.
2. Data migration or backend model.
3. API/service logic.
4. UI component/page integration.
5. Tests and release readiness.
