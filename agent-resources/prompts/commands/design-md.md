# DESIGN.md Command Prompt

Generate a Stitch-compatible DESIGN.md from existing frontend source code or user design intent.

Read repository evidence first: package.json, routes, pages, components, theme files, CSS variables, Tailwind config, design tokens, Storybook files, and existing docs.

Required structure: visual theme, color palette and roles, typography, component stylings, layout principles, depth/elevation, do/don't rules, responsive behavior, agent prompt guide, source evidence and assumptions, next command.

Use the document as a portable design source of truth for Copilot, OpenCode, Stitch-style workflows, Claude Code, Codex, and other AI agents.
