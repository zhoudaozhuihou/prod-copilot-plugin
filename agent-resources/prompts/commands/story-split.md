# Story Split Command Prompt

Split PRD, feature design, or a broad request into Ralph-sized stories.

## Rules

- One story per fresh agent context.
- Dependency order first, business value second.
- Split too-large stories until each has a focused implementation boundary.
- Each story has ID, title, type, description, acceptance criteria, dependency, and validation evidence.
- Do not combine unrelated frontend, backend, data, and release work into one story.
