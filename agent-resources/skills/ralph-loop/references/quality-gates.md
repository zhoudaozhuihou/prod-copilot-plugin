# Ralph Quality Gates

Every completed story needs evidence.

## Code stories

- Typecheck passes.
- Lint passes.
- Unit or integration tests pass when relevant.

## UI stories

- Browser or manual visual verification is completed.
- Loading, error, empty, disabled, and permission states are checked when applicable.

## Data stories

- DQ check SQL is provided.
- Reconciliation check is provided.
- Partition/watermark/backfill impact is documented.

## Release stories

- Rollback path exists.
- Monitoring and runbook updates exist.
