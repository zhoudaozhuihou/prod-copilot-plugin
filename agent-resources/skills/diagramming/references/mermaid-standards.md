# Mermaid Standards

- Use `flowchart TD` or `flowchart LR` for architecture, user flows, DQ, reconciliation, release, and runbook processes.
- Use `sequenceDiagram` for request/response and interaction timing.
- Use `journey` for user journey maps.
- Use `stateDiagram-v2` for UI, workflow, or lifecycle states.
- Use `erDiagram` for relational models.
- Keep labels business-readable.
- Avoid overly dense diagrams; split diagrams when more than 12-15 nodes.
- Add assumptions below each diagram instead of hiding uncertainty inside the diagram.
