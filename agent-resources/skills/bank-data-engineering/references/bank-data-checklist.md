# Bank Data Engineering Checklist

## Data contract

Owner, SLA, grain, primary/unique keys, nullable fields, enum rules, partition strategy, sensitive classification, compatibility, downstream consumers.

## STTM

Source field, target field, type conversion, transformation rule, filter, join rule, default/null handling, DQ check, reconciliation check, owner.

## Controls

DQ, reconciliation, field lineage, privacy/masking, scheduler, backfill, release gate, runbook, incident escalation.
