# Ordered Product Development Workflow

1. /init — initialize configuration and folders.
2. /scan — inspect repository context.
3. /plan — choose the minimal necessary delivery path.
4. /brainstorm — explore opportunities and alternatives.
5. /feature — converge into a feature design.
6. /prd — write engineering-ready requirements.
7. /journey — validate user path and friction.
8. /api — define contract-first API boundaries.
9. /frontend — design or implement frontend work.
10. /backend, /springboot, /python — design or implement backend work.
11. /data, /dbschema, /sql, /pipeline — design or implement data work.
12. /task — split into Jira-ready delivery tasks.
13. /test — generate test plan and test cases.
14. /quality — define quality gates.
15. /review — perform enterprise review.
16. /diff — update artifacts from git changes.
17. /release — generate release readiness pack.
